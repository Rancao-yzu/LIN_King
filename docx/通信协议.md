# LIN 通讯协议规范（沙发项目）LIN_sofa_V1.2_20260618

## 指令内容

指令包含 **PID段**（1字节）和 **数据段**（8字节）。

### 1. 指令长度

- **PID段**：  
  - LIN_PACK.ID 占 6 bit  
  - LIN_PACK.P0 和 LIN_PACK.P1 各占 1 bit  
  - 共 **1 字节**

- **数据段**：  
  - LIN_PACK.DATA[] 共 **8 字节**，用于传输具体指令内容。

---

### 2. LIN 通讯背景

- **主节点** 对 **从节点** 发送**控制帧**，从节点执行动作，**不回复**。
- **主节点** 对 **从节点** 发送**状态帧**，从节点收到后**必须回复**响应帧。
- 从节点不会主动上报数据，仅响应状态帧请求。

- **当前节点角色**：
  - 主节点：上位机
  - 从节点：电机、雷达

---

### 3. 上位机通讯内容定义

#### 3.1 控制帧（上位机 → 从节点）

- **防丢机制**：每条控制指令连续发送 **5 次**，周期 **100ms**。
- 帧 ID 定义：
  - 电机控制帧 ID：`0x01`
  - 雷达控制帧 ID：`0x03`

**帧格式**：

| PID（1字节） | DATA[0] | DATA[1] | DATA[2] | DATA[3] | DATA[4] | DATA[5] | DATA[6] | DATA[7] |
|--------------|---------|---------|---------|---------|---------|---------|---------|---------|
| ID（6bit）+ P0/P1 | 见下表 | 见下表 | 见下表 | RES | RES | RES | RES | RES |

**DATA[0] ~ DATA[2] 定义**：

| 字节 | 取值（hex） | 含义 |
|------|-------------|------|
| **DATA[0]** <br>（电机自学习状态） | `0x00` | 默认值 |
| | `0x01` | 电机进行自学习 |
| | `0x02` | 电机**不**进行自学习 |
| | `0x03` | RES |
| **DATA[1]** <br>（算法干扰前电机状态） | `0x00` | 默认值 |
| | `0x01` | 电机转动伸出 |
| | `0x02` | 电机转动收回 |
| | `0x03` | 电机停止转动 |
| **DATA[2]** <br>（有无障碍物） | `0x00` | 默认值（算法未就绪或雷达故障） |
| | `0x01` | 无障碍物 |
| | `0x02` | 有障碍物 |
| | `0x03` | RES |

> **说明**：
> - 请求电机自学习时，`DATA[1]` 和 `DATA[2]` 应置为 `0x00`。
> - 控制电机伸出/收回/停止时，`DATA[0]` 和 `DATA[2]` 应置为 `0x00`。
> - 当雷达算法执行中或雷达故障时，`DATA[2]` 须设为 `0x00`（默认值）。
> - `DATA[3]~DATA[7]` 为保留字节（RES），填充 `0x00`。

---

#### 3.2 状态帧（上位机 → 从节点，仅帧头）

- **用途**：请求从节点回复当前状态（行程、故障等）。
- **帧周期**：100ms（持续请求电机行程）。
- 帧 ID 定义：
  - 电机状态帧 ID：`0x02`
  - 雷达状态帧 ID：`0x04`
- **防丢机制**：同控制帧（连续发送 5 次，周期 100ms）。

**帧格式**：只发送 PID 段（包含 ID + P0/P1），不携带数据字节（或发送 8 个 `0x00`，具体待调试确认）。

| PID（1字节） |
|--------------|
| ID（6bit）+ P0/P1 |

---

### 4. 电机响应通讯内容

#### 4.1 控制帧回复

电机**不回复**控制帧。

#### 4.2 状态帧回复（电机 → 上位机）

- **帧 ID**：`0x02`（与请求帧 ID 相同）
- **数据长度**：8 字节

**回复帧格式**：

| PID | DATA[0] | DATA[1] | DATA[2] | DATA[3] | DATA[4] | DATA[5] | DATA[6] | DATA[7] |
|-----|---------|---------|---------|---------|---------|---------|---------|---------|
| 0x02+P | 自学习标志 | 当前位置（低字节） | 当前位置（高字节） | 最大行程 | 故障帧 | RES | RES | RES |

**各字节详细定义**：

| 字节 | 取值 | 说明 |
|------|------|------|
| **DATA[0]** <br>自学习完成标志 | `0x00` | 自学习未完成 |
| | `0x01` | 自学习已完成 |
| **DATA[1~2]** <br>电机当前位置 | int16（低字节在前） | 单位：**个**（行程计数），无物理单位 |
| **DATA[3]** <br>电机最大行程 | 待定义 | 自学习成功后更新，学习前为默认值（如 `0x00`） |
| **DATA[4]** <br>故障帧 | 待细化 | 后续定义具体故障码 |
| **DATA[5~7]** | RES | 保留，填充 `0x00` |

> **注意**：电机启动后需先完成自学习以标记行程起点和终点，成功后 `DATA[0]` 置 `0x01`，并更新最大行程值。

---

### 5. 雷达响应通讯内容

#### 5.1 控制帧回复

雷达**不回复**控制帧。

#### 5.2 状态帧回复（雷达 → 上位机）

- **帧 ID**：`0x04`（与请求帧 ID 相同）
- **数据长度**：8 字节

**回复帧格式**：

| PID | DATA[0] | DATA[1] | DATA[2] | DATA[3] | DATA[4] | DATA[5] | DATA[6] | DATA[7] |
|-----|---------|---------|---------|---------|---------|---------|---------|---------|
| 0x04+P | 障碍物标志 | 故障帧 | RES | RES | RES | RES | RES | RES |

**各字节详细定义**：

| 字节 | 取值 | 说明 |
|------|------|------|
| **DATA[0]** <br>有无障碍物 | `0x00` | 默认值（算法未就绪或雷达故障） |
| | `0x01` | 无障碍物 |
| | `0x02` | 有障碍物 |
| | `0x03` | RES |
| **DATA[1]** <br>故障帧 | `0x00` | 无故障 |
| | `0x01` | 遮挡故障 |
| | `0x02~0xFF` | RES（预留：超时、版本号、电压/温度、标定、内部故障等） |
| **DATA[2~7]** | RES | 保留，填充 `0x00` |

---

### 6. 通讯协议

#### 6.1 P0、P1 奇偶校验位

- **作用**：对 6 位 ID 进行奇偶校验，接收端快速判断 PID 是否传输错误。
- **计算规则**（标准 LIN 公式）：

```
P0 = ID0 ⊕ ID1 ⊕ ID2 ⊕ ID3 ⊕ ID4
P1 = ~(ID1 ⊕ ID2 ⊕ ID3 ⊕ ID4 ⊕ ID5)
```

其中 `ID0` 为最低位，`ID5` 为最高位。  
只有校验通过，从节点才会响应帧，否则忽略。

#### 6.2 校验和计算（增强型）

- **范围**：`PID` 字节 + 全部 8 个数据字节。
- **计算步骤**：
  1. 初始化累加和 `checksum = 0`。
  2. 按顺序将 PID 和 DATA[0]~DATA[7] 逐个字节加到 `checksum`。
  3. 每次累加后，若超过 8 位（>0xFF），将溢出进位加到低 8 位：  
     `checksum = (checksum & 0xFF) + (checksum >> 8)`。
  4. 全部字节加完后，最终校验和 = `0xFF - checksum`（结果占 1 字节）。

- **示例**（PID = 0x41，数据全 0）：
  - 累加：`checksum = 0x41 + 0 + ... + 0 = 0x41`
  - 无进位，校验和 = `0xFF - 0x41 = 0xBE`

> 校验和放在帧的最后 1 个字节（即第 9 个字节，紧随 8 个数据字节之后）。

## 附录：帧周期总结

| 帧类型 | 发送方 → 接收方 | 周期 | 备注 |
|--------|----------------|------|------|
| 控制帧（电机自学习/动作/障碍物） | 上位机 → 电机/雷达 | 100ms，连续5次 | 防丢机制 |
| 状态帧（请求行程） | 上位机 → 电机 | 100ms（持续） | 电机回复当前位置 |
| 状态帧（请求障碍物/故障） | 上位机 → 雷达 | 100ms（持续） | 雷达回复障碍物和故障 |
